# ملخّص المشروع — PROJECT_SUMMARY

> الملف ده هو المرجع الشامل للمشروع. أي حد جديد يقراه يفهم: المشروع ده إيه، بيشتغل إزاي، اتعمل منه إيه، وإيه القادم. يتم تحديثه مع كل خطوة.

---

## ١. الفكرة

منظومة برمجية تعتمد على عدة وكلاء ذكاء اصطناعي (`multi-agent`)، تتيح بناء تطبيقات ومواقع من مواصفات مكتوبة، عبر لوحة تحكم على الويب — دون الحاجة لخبرة برمجية عميقة من المستخدم.

## ٢. المبادئ الأساسية

- كل الأدوات مفتوحة المصدر و `self-hosted` ومجانية.
- التكلفة الوحيدة: استدعاءات `API` لمزوّد النماذج.
- تعمل على `Proxmox VM` واحدة.
- كل الأسرار (مفاتيح، كلمات مرور) في مكان واحد فقط: `Infisical`.
- الإدارة من المتصفح، بأقل استخدام ممكن للترمينال.

## ٣. المعمارية

| المكوّن | الأداة |
|---|---|
| البنية التحتية | `Ubuntu` + `Docker` + `Portainer` |
| إدارة الأسرار | `Infisical` |
| بوابة النماذج | `LiteLLM` |
| المبرمج | `OpenHands` |
| تنسيق الوكلاء | `CrewAI` |
| لوحة البداية | صفحة موحّدة |

## ٤. الأدوار الخمسة (خط العمل)

1. **الباحث / المحلل** — يبحث في الفكرة، يجمع المعلومات ومصادرها، ويناقش المستخدم للوصول لفكرة قابلة للتنفيذ.
2. **المصمم** — يحوّل الفكرة إلى `mockup` فعلي للواجهات دون دخول في كود معقّد.
3. **المخطّط (المهندس المعماري)** — يأخذ الفكرة والـ `mockup` ويضع خطة `frontend`/`backend` و `todo` مرتّبة.
4. **المبرمج** (`OpenHands`) — ينفّذ الكود بناءً على الخطة، خطوة بخطوة.
5. **المراجع** — يفحص الكود أمنيًا ووظيفيًا، ويُبلّغ عن الفجوات.

> ملاحظة: المراجعة النهائية للأمان تبقى بشرية. الوكيل المراجع مساعد وليس بديلًا.
